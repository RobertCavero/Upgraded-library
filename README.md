# Upgraded Library 📚

O **Upgraded Library** é um sistema de gerenciamento de biblioteca digital. O projeto visa oferecer uma experiência eficiente e estruturada para que usuários possam explorar, organizar e acompanhar seus interesses literários por meio de recursos tecnológicos modernos.

A plataforma centraliza funcionalidades de pesquisa, salvamento de favoritos e acompanhamento de progresso de leitura em uma única aplicação.

---

## 🚀 Funcionalidades

O sistema foi projetado para atender leitores, estudantes e pesquisadores, oferecendo os seguintes recursos:

*   **Autenticação de Usuários:** Cadastro e login de usuários para acesso personalizado.
*   **Catálogo Digital:** Exibição de livros disponíveis com informações sobre sua disponibilidade.
*   **Pesquisa Personalizada:** Busca de obras por tema, título ou autor.
*   **Gestão de Favoritos:** Permite adicionar e remover livros de uma lista de favoritos.
*   **Status de Leitura:** Classificação de livros em categorias como **"Lendo"**, **"Lido"** e **"Quero Ler"**.
*   **Estatísticas Pessoais:** Exibição de dados sobre o perfil e progresso de leitura do usuário.

---

## 🛠️ Tecnologias Utilizadas

O projeto utiliza uma stack focada em **performance e escalabilidade**:

*   **Front-end:** [React](https://reactjs.org/) e JavaScript, com foco em uma interface dinâmica e responsiva.
*   **Back-end:** [Node.js](https://nodejs.org/), responsável pela lógica de negócio e autenticação.
*   **ORM:** [Prisma ORM](https://www.prisma.io/) para modelagem e manipulação segura de dados.
*   **Banco de Dados:** [Neon Database](https://neon.tech/) (PostgreSQL), operando como solução de armazenamento em nuvem.
*   **Hospedagem:**
    *   Front-end: [Vercel](https://vercel.com/).
    *   Back-end (API): [Render](https://render.com/).

---

## 🏗️ Arquitetura e Metodologia

O sistema segue o modelo **Cliente-Servidor**, garantindo uma separação clara entre a interface do usuário, a lógica de negócio e o banco de dados.

*   **Componentização:** Interface estruturada em componentes reutilizáveis no React.
*   **API REST:** Comunicação via requisições HTTP para atualizações em tempo real.
*   **Metodologia Ágil:** O desenvolvimento foi baseado no framework **Scrum**, com ciclos iterativos e incrementais.
